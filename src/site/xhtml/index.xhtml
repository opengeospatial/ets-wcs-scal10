WCS 2.0.1 Scaling Extension (12-039) compliance test suite

Scope

This Executable Test Suite (ETS) verifies the conformance of WCS 2.0 Scaling Extension. The services should be accessed through WCS GetCapabilities, DescribeCoverage, and GetCoverage using the HTTP KVP, POST, and SOAP protocol binding. The GetCoverage response should be provided as a GML Coverage instance document.

Test requirements

The documents listed below stipulate requirements that must be satisfied by a conforming application schema.

  • [OWS Common] OGC Web Services Common Standard (OGC 06-121r9)
  • [WCS Core] OGC® WCS 2.0 Interface Standard - Core (OGC 09-110r3)
  • [WCS Core Corrigendum] OGC® WCS 2.0 Interface Standard - Core: Corrigendum (09-110r4)
  • [GMLCOV] OGC® GML Application Schema - Coverages (09-146r2)
  • [WCS KVP] OGC® Web Coverage Service 2.0 Interface Standard - KVP Protocol Binding Extension - Corrigendum (OGC 09-147r3)
  • [WCS POST] OGC® Web Coverage Service 2.0 Interface Standard - POST Protocol Binding Extension - Corrigendum (09-148r1)
  • [WCS SOAP] OGC® Web Coverage Service 2.0 Interface Standard - SOAP Protocol Binding Extension - Corrigendum (09-149r1)
  • [WCS SCAL] OGC® Web Coverage Service 2.0 Interface Standard - Scaling Extension (12-039)
  • Compliance Test Language (CTL) 0.5

General requirements for the implementation:

  1. At least one protocol implementation should be available and advertised in the GetCapabilities response.
  2. The implementation provides at least two coverages.
  3. The first coverage offering should be a RectifiedGridCoverage with at least 2 dimensions, while the second coverage can be any coverage.
  4. Do not use CoverageId_Bogus as a coverage ID, format_bogus as a format name, mediatype_bogus for the mediatype parameter, or dimension_bogus as an axis name.
  5. The offered coverage must be implemented according to the GMLCov specification (09-146r2).
